### 5.4 Page Speed Targets (Core Web Vitals)

| Metric | Target | How to Achieve |
|--------|--------|----------------|
| LCP (Largest Contentful Paint) | Under 2.5 seconds | Optimize hero image, preload fonts, use CDN |
| FID (First Input Delay) | Under 100ms | Defer non-critical JS, minimize third-party scripts |
| CLS (Cumulative Layout Shift) | Under 0.1 | Set explicit width/height on all images, reserve space for embeds |
| TTFB (Time to First Byte) | Under 600ms | Fast hosting, server-side caching, CDN |
| Total Page Size | Under 2MB | Compress images, minify CSS/JS, remove unused code |
| Speed Index | Under 3 seconds | Critical CSS inline, defer below-fold content |

**Action Items:**
- [ ] Enable Gzip/Brotli compression on server
- [ ] Minify all CSS and JavaScript files
- [ ] Use a CDN (Cloudflare free tier at minimum)
- [ ] Implement browser caching headers (1 year for static assets)
- [ ] Inline critical above-the-fold CSS
- [ ] Defer all non-essential JavaScript
- [ ] Convert all images to WebP format
- [ ] Lazy load all images below the fold
- [ ] Preload the hero image, primary font, and logo
- [ ] Remove unused CSS (especially from frameworks)
- [ ] Test monthly with PageSpeed Insights and Lighthouse

### 5.5 Mobile-First Indexing

Google indexes the mobile version of the site first. Non-negotiable requirements:

- [ ] Responsive design (no separate mobile site)
- [ ] Viewport meta tag: `<meta name="viewport" content="width=device-width, initial-scale=1">`
- [ ] Tap targets at least 48x48px with 8px spacing
- [ ] Font size minimum 16px for body text
- [ ] No horizontal scrolling at any viewport
- [ ] Phone number is a clickable `tel:` link everywhere: `<a href="tel:+16784852303">678.485.2303</a>`
- [ ] Forms are easy to fill on mobile (large inputs, proper input types)
- [ ] No intrusive interstitials (popups covering content on mobile)
- [ ] Same content on mobile as desktop (no hidden content)
- [ ] Test with Google Mobile-Friendly Test tool
